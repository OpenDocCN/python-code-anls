
# `graphrag\packages\graphrag\graphrag\prompt_tune\prompt\language.py` 详细设计文档

该文件定义了一个用于语言检测的微调提示模板，通过预定义的格式指导语言模型识别输入文本的主要语言，仅返回语言名称如'English'、'Spanish'等。

## 整体流程

```mermaid
graph TD
    A[加载提示模板] --> B[格式化模板: 替换{input_text}占位符]
B --> C[将格式化后的提示发送给语言模型]
C --> D[模型返回语言名称]
D --> E[解析并返回结果]
```

## 类结构

```
无类层次结构 - 仅包含单个全局常量
```

## 关键组件





### DETECT_LANGUAGE_PROMPT

全局提示模板变量，用于语言检测任务的少样本提示（Few-shot Prompting）框架。

### 潜在的技术债务或优化空间

- **硬编码提示模板**：提示内容直接写死在代码中，缺乏灵活性
- **缺乏国际化支持**：提示模板目前仅支持英文，无法直接扩展到多语言场景
- **无版本管理**：提示模板无版本控制机制，难以追踪历史变更
- **缺少输入验证**：未对 `{input_text}` 占位符替换进行安全检查
- **无错误处理机制**：模板渲染失败时缺乏降级策略

### 其它项目

- **设计目标**：通过系统提示词引导大语言模型完成语言分类任务
- **约束条件**：模型输出被限制为仅返回语言名称，需确保回复格式一致
- **输入规范**：待检测文本通过 `{input_text}` 占位符注入
- **外部依赖**：无外部依赖，仅为纯字符串常量定义



## 问题及建议



### 已知问题

- **硬编码提示模板**：提示模板直接硬编码为常量，缺乏灵活配置，无法在不修改代码的情况下调整提示策略
- **缺少输入验证**：未对 `{input_text}` 占位符的输入进行长度、内容或安全性验证，可能导致提示注入或格式问题
- **语言示例不完整**：示例仅列出4种语言（English, Spanish, Japanese, Portuguese），但实际应用可能需要支持更多语言，且示例与"among others"暗示的灵活性不一致
- **缺乏可扩展性**：作为单一常量，无法支持多语言变体、版本化提示或A/B测试场景
- **无错误处理机制**：调用方使用此常量时缺乏错误处理指导或异常定义
- **字符串格式化风险**：直接使用字符串替换 `{input_text}`，未对特殊字符进行转义，可能导致提示结构破坏

### 优化建议

- **外部化配置**：将提示模板移至配置文件（如 YAML/JSON）或环境变量，支持动态调整
- **添加输入验证**：在模板使用处增加输入验证逻辑，如最大长度限制、字符过滤等
- **参数化语言列表**：将示例语言列表抽取为配置项，支持动态扩展
- **版本管理**：为提示模板添加版本号，支持提示迭代和回滚
- **结构化替代方案**：考虑使用模板引擎（Jinja2）或专门的 Prompt Library 管理复杂提示
- **添加文档注释**：补充模块级和常量级文档，说明使用场景、限制和示例

## 其它





### 设计目标与约束

本模块的设计目标是为语言检测功能提供标准化的提示模板，支持通过占位符 `{input_text}` 动态插入待检测文本。约束条件包括：仅作为静态字符串常量使用，不包含任何运行时逻辑；提示内容限定为英文，回复内容强制为语言名称。

### 错误处理与异常设计

由于本模块仅定义静态常量，不涉及任何运行时逻辑，因此不设计错误处理与异常机制。使用方在填充占位符时需自行保证输入有效性。

### 数据流与状态机

不适用。本模块为无状态模块，仅提供数据定义，不涉及数据流处理或状态管理。

### 外部依赖与接口契约

本模块无外部依赖。接口契约方面，`DETECT_LANGUAGE_PROMPT` 为字符串类型常量，使用方需使用字符串的 `format()` 方法或 `str.format_map()` 方法填充 `{input_text}` 占位符，填充后的字符串将作为大语言模型的输入。

### 性能考虑

本模块为纯静态定义，不涉及运行时计算，无性能瓶颈。字符串模板在程序初始化时即已加载至内存。

### 安全性考虑

提示内容本身不涉及敏感信息或个人数据。但使用方需注意：若将包含用户敏感信息的文本填充至该提示并提交至大语言模型，需确保符合相关隐私保护法规。

### 配置管理

本模块采用代码内嵌方式管理配置，作为常量定义直接使用。若未来需要支持多语言或可配置提示内容，可考虑迁移至独立的配置文件或环境变量。

### 测试策略

建议测试策略包括：验证 `DETECT_LANGUAGE_PROMPT` 字符串非空；验证字符串包含 `{input_text}` 占位符；验证字符串格式化后的输出符合预期格式。

### 版本兼容性

本模块使用标准的 Python 3 字符串语法，兼容 Python 3.6 及以上版本。

### 国际化/本地化

当前提示文本为英文，仅支持单一语言。若需支持多语言检测场景（如检测中文、日文等），需要扩展为多语言版本的提示模板字典。
