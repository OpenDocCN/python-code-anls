
# `graphrag\packages\graphrag\graphrag\prompts\query\global_search_knowledge_system_prompt.py` 详细设计文档

这是一个全局搜索系统的提示词配置文件，定义了用于指导大语言模型在生成响应时如何标注真实世界知识的指令模板，确保模型能够区分数据集内的知识和需要验证的真实世界信息。

## 整体流程

```mermaid
graph TD
A[系统启动] --> B[加载system_prompts.py]
B --> C[获取GENERAL_KNOWLEDGE_INSTRUCTION]
C --> D[将指令注入到LLM提示词中]
D --> E[LLM生成响应]
E --> F{响应包含外部知识?}
F -- 是 --> G[标注[LLM: verify]标签]
F -- 否 --> H[直接输出响应]
G --> H
```

### 核心功能概述

该代码定义了一个全局系统提示常量，用于指导大语言模型在生成回答时，可以引入外部真实世界知识，但需要使用 `[LLM: verify]` 标签进行标注，以便用户区分数据集内知识和需要验证的外部知识。

### 文件整体运行流程

该文件为配置型模块，不包含可执行逻辑。模块被导入时，仅完成常量定义，供其他模块通过 `from ... import` 或 `import ...` 方式引用使用。

### 全局变量详情

#### GENERAL_KNOWLEDGE_INSTRUCTION

- **类型**: `str`
- **描述**: 系统级提示词字符串，定义了大语言模型在使用外部真实世界知识时的标注规范，要求必须使用 `[LLM: verify]` 标签进行标注。

### 关键组件信息

#### 1. GENERAL_KNOWLEDGE_INSTRUCTION 提示模板

该系统提示模板定义了 Global Search 功能中对于外部知识的处理规范，确保模型输出的可追溯性和可信度。

### 潜在技术债务与优化空间

1. **配置外部化不足**: 提示词直接硬编码在代码中，不利于非技术人员调整和维护，考虑迁移至独立的配置文件或数据库。

## 其它




### 设计目标与约束

该代码定义了一个用于Global Search系统的系统提示词常量，旨在指导大型语言模型（LLM）在生成回答时如何处理和标注数据集外的真实世界知识。设计约束包括：提示词内容必须使用英文，标注标签格式必须为"[LLM: verify]"。

### 错误处理与异常设计

由于该代码仅包含静态字符串常量，不涉及运行时错误处理。若该常量未被正确定义或导入失败，使用方Python模块将抛出`NameError`或`ImportError`。建议在调用处进行异常捕获或依赖注入验证。

### 数据流与状态机

该常量作为只读配置数据被外部模块导入并传递给LLM调用流程。无状态机设计，数据流为：定义常量 → 导入模块 → 传递至LLM系统提示词 → 影响生成内容。

### 外部依赖与接口契约

该常量无外部依赖，仅为纯Python字符串定义。接口契约为：调用方需导入`GENERAL_KNOWLEDGE_INSTRUCTION`并将其作为system prompt的组成部分传递给LLM API。

### 安全性考虑

该常量不涉及敏感信息，但包含标记"[LLM: verify]"用于提示LLM区分事实与推测内容。安全性建议：确保下游LLM调用时正确渲染该标记，避免被恶意用户输入覆盖。

### 可扩展性设计

当前设计为单一常量，扩展性有限。未来可考虑：1）将其重构为配置类，支持多语言；2）支持从外部配置文件加载；3）添加版本号管理。
