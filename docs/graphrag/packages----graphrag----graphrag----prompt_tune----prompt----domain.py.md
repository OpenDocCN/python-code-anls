
# `graphrag\packages\graphrag\graphrag\prompt_tune\prompt\domain.py` 详细设计文档

这是一个用于文本领域分类的fine-tuning提示词模板，通过预定义的指令和示例，帮助AI模型理解如何分析输入文本并生成描述性领域标签，如'Social studies'、'Algorithmic analysis'、'Medical science'等。

## 关键组件




### 提示模板组件 (Prompt Template)

这是一个多行字符串常量，定义了一个用于文本领域分类的提示模板，包含角色定义、任务说明、示例格式化和输入占位符。

这是一个字符串格式化占位符，用于在运行时动态插入待分析的文本内容，实现提示模板的动态化。

## 问题及建议





### 已知问题

-   **缺乏输入验证机制**：代码中的 `{input_text}` 占位符没有对应的输入验证逻辑，可能导致空值、恶意输入或格式异常的文本直接传入下游模型，造成潜在的 Prompt 注入攻击风险
-   **硬编码的 Prompt 字符串**：Prompt 内容被直接硬编码在代码中，无法在不修改源代码的情况下调整提示词内容，缺乏灵活性和可配置性
-   **缺少类型注解**：全局变量 `GENERATE_DOMAIN_PROMPT` 缺少类型注解，降低了代码的可读性和静态分析工具的检查能力
-   **无错误处理机制**：当模板替换失败或占位符未正确填充时，代码没有任何错误处理或异常抛出逻辑
-   **单语言局限性**：Prompt 内容仅支持英文，缺乏国际化支持，无法直接应用于多语言场景
-   **Prompt 版本管理缺失**：作为重要的业务提示词，没有版本控制或变更记录，难以追踪和回滚
-   **文档注释不足**：仅有模块级 docstring，缺少对 Prompt 用途、适用场景、预期输入输出格式的详细说明

### 优化建议

-   **添加输入验证函数**：创建专门的验证函数，检查输入文本是否为空、是否包含危险字符或模式，并提供清晰的错误信息
-   **配置化管理**：将 Prompt 模板提取至独立的配置文件（如 YAML 或 JSON），支持运行时动态加载和切换不同版本的提示词
-   **增加类型注解**：为全局变量添加 `str` 类型注解，提升代码可维护性
-   **实现模板替换安全机制**：使用安全的字符串格式化方法（如 `str.format()` 的严格模式或正则表达式替换），并添加占位符完整性检查
-   **支持国际化**：引入 i18n 框架或设计多语言 Prompt 模板机制，支持根据 locale 动态加载对应语言的提示词
-   **补充文档注释**：在 docstring 中详细说明 Prompt 的设计意图、输入输出规范、示例用例以及版本变更历史
-   **添加单元测试**：为输入验证、模板替换等功能编写单元测试，确保边界情况和异常场景的正确处理



## 其它





### 设计目标与约束

该提示模板用于文本领域分类任务，旨在帮助大语言模型将输入文本归类到适当的领域类别（如"Social studies"、"Algorithmic analysis"、"Medical science"等）。设计约束包括：仅支持英文领域术语输出，输入文本长度应控制在合理范围内（建议不超过10000字符），输出格式固定为单行领域名称。

### 输入验证规范

**验证规则**：
- input_text必须为非空字符串
- 文本长度建议不超过10000字符
- 不应包含控制字符或特殊格式指令

**边界情况处理**：
- 空文本：返回"General"领域
- 过长文本：建议截断或返回"Unknown"领域
- 非文本输入：应抛出TypeError异常

### 输出格式规范

**期望输出格式**：
- 单行纯文本输出
- 领域名称首字母大写
- 不包含额外标点符号或解释文本

**示例输出**：
```
Medical science
```

### 使用示例与调用模式

**基础调用示例**：
```python
prompt = GENERATE_DOMAIN_PROMPT.format(input_text="The heart is a muscular organ...")
# 输出: "You are an intelligent assistant...\n\nText: The heart is a muscular organ...\nDomain:"
```

**集成到LLM调用流程**：
1. 获取用户输入文本
2. 使用format()方法填充占位符
3. 将生成的提示发送给语言模型
4. 解析模型返回的领域分类结果

- 警告：输入文本可能包含机密或个人信息
- 建议：在生产环境中实施输入过滤和脱敏处理
- 建议：记录日志时排除原始输入文本

### 错误处理与异常设计

**异常类型**：
- ValueError：输入为空或格式不正确
- TypeError：输入类型非字符串
- KeyError：模板占位符不匹配

**处理策略**：
- 应该在调用方进行参数验证
- 提供有意义的错误消息帮助调试
