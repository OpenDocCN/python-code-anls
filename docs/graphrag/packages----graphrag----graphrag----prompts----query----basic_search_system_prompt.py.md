
# `graphrag\packages\graphrag\graphrag\prompts\query\basic_search_system_prompt.py` 详细设计文档

该文件定义了基础搜索系统提示词模板，用于指导AI助手如何根据数据表中的信息生成符合目标长度和格式的响应，包含角色定义、目标说明、数据引用规则和响应格式要求。

## 整体流程

```mermaid
graph TD
A[加载提示词模板] --> B[替换{response_type}占位符]
B --> C[替换{context_data}占位符]
C --> D[生成最终系统提示词]
```

## 类结构

```
该文件为纯配置文件，无类层次结构
仅包含一个全局字符串变量 BASIC_SEARCH_SYSTEM_PROMPT
```

## 全局变量及字段


### `BASIC_SEARCH_SYSTEM_PROMPT`
    
用于AI助手响应数据表查询的系统提示词模板，包含角色定义、响应格式要求和数据引用规范

类型：`str`

## 关键组件




### 角色定义模块

定义AI助手为帮助用户回答关于表格数据的问题的定位

### 目标模块

指导生成符合目标长度和格式的响应，并优先使用输入数据表中的数据

### 数据引用规范

定义数据引用的标准格式，包括记录ID列表（最多5条）和"+more"扩展表示法

### 响应类型占位符

{response_type}占位符，用于动态指定响应的长度和格式要求

### 上下文数据占位符

{context_data}占位符，用于插入待查询的数据表格内容

### Markdown格式化指导

指导响应使用markdown格式并根据长度和格式适当添加章节和评论

### 未知答案处理策略

明确指示当信息不足时应明确说明不知道，而非编造内容

### 多源数据引用示例

提供具体的数据引用格式示例，展示如何在响应中标注数据来源


## 问题及建议




### 已知问题

-   **提示模板内容重复**：模板中"Goal"和"Target response length and format"部分重复出现两次，增加了维护成本，容易导致不一致修改
-   **缺乏文档说明**：没有模块级或常量级的文档字符串（docstring），难以理解该常量的用途和使用场景
-   **硬编码字符串**：所有提示内容完全硬编码，缺乏灵活的配置机制，修改任何提示内容都需要直接修改代码
-   **占位符无验证**：模板中的`{response_type}`和`{context_data}`占位符没有任何运行时验证，使用方可能传入错误格式导致提示失效
-   **无类型注解**：常量缺乏类型注解，不利于静态分析和IDE自动补全
-   **魔法数字限制**：引用记录ID时限制"不超过5个"的规则硬编码在提示文本中，若需调整需修改字符串

### 优化建议

-   **消除重复内容**：将重复的"Goal"和"Target response length and format"部分合并为一次，简化模板结构
-   **添加文档字符串**：为模块添加docstring，说明该文件用于Basic Search的系统提示，并描述常量用途
-   **模块化拆分**：将提示模板拆分为多个独立的部分常量，通过拼接函数组装，提高可维护性
-   **配置化设计**：考虑将提示的各个部分（如角色定义、格式要求、引用规则）提取为独立配置，支持灵活组合
-   **添加验证逻辑**：在模板使用处添加对`response_type`和`context_data`参数的类型和格式验证
-   **提取常量**：将"不超过5个记录ID"等业务规则提取为配置常量，便于未来调整
-   **类型注解**：为常量添加类型注解，如`BASIC_SEARCH_SYSTEM_PROMPT: str`


## 其它





### 设计目标与约束

该代码定义了一个基础搜索场景下的系统提示词模板，用于指导AI助手回答用户关于表格数据的问题。设计目标是确保AI生成的响应准确引用数据源、控制响应长度、并遵循特定的格式规范。约束条件包括：不虚构信息、数据引用不超过5条记录ID、必须使用markdown格式化响应。

### 错误处理与异常设计

由于该代码仅为静态字符串常量，不涉及运行时错误处理。若模板中占位符（如{response_type}、{context_data}）未被正确替换，应由调用方负责处理。调用方需确保传入参数非空且格式正确，否则可能导致提示词失效或生成不符合预期的响应。

### 数据流与状态机

该代码本身不涉及数据流处理或状态机逻辑。数据流发生在调用方：调用方准备上下文数据（context_data）和响应类型（response_type），将这些数据填入模板的占位符位置，生成最终的系统提示词，然后发送给语言模型。整个过程为单向数据流，无状态转换。

### 外部依赖与接口契约

该代码无外部依赖，仅为纯字符串定义。接口契约方面，调用方需满足以下要求：必须提供response_type参数指定响应格式（如"简短"、"详细"等），必须提供context_data参数包含表格数据，其他占位符应根据实际需求填充。调用方应使用str.format()、f-string或模板引擎进行占位符替换。

### 安全性考虑

该代码为静态提示词模板，安全性风险较低。但调用方在填充context_data时应注意：避免将敏感数据未经脱敏直接放入提示词、确保数据来源合法、在用户界面中明确标注数据引用来源。提示词本身不包含恶意指令，安全性主要取决于调用方的数据处理方式。

### 性能要求

该代码为字符串常量，内存占用极低，访问速度仅取决于字符串复制效率。性能优化主要依赖于调用方：避免重复创建相同提示词实例、考虑缓存替换后的完整提示词、在高并发场景下使用预编译模板。

### 使用示例

```python
# 基本使用示例
response_type = "简短"
context_data = "表格数据..."

final_prompt = BASIC_SEARCH_SYSTEM_PROMPT.format(
    response_type=response_type,
    context_data=context_data
)
# 将final_prompt发送给语言模型
```
