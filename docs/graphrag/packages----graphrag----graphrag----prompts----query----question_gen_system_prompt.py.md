
# `graphrag\packages\graphrag\graphrag\prompts\query\question_gen_system_prompt.py` 详细设计文档

这是一个问题生成系统的提示词模板文件，定义了用于指导AI模型根据表格数据生成候选问题的系统提示词，包含角色定义、数据来源、目标说明和示例问题等部分。

## 整体流程

```mermaid
graph TD
A[开始] --> B[定义QUESTION_SYSTEM_PROMPT模板]
B --> C{需要生成问题时}
C --> D[填充{question_count}变量]
D --> E[填充{context_data}变量]
E --> F[将完整提示词传给LLM]
F --> G[LLM生成候选问题列表]
```

无类层次结构（纯配置文件）

## 关键组件





### QUESTION_SYSTEM_PROMPT

系统级提示模板，定义了AI助手生成问题列表时的角色定位、数据输入格式、生成目标和示例规范。

### 模板变量 {question_count}

动态参数，用于指定生成的问题数量，在实际调用时由外部传入具体数值。

### 模板变量 {context_data}

动态参数，用于填充实际的数据表格内容，作为生成问题的数据来源。

### 角色定义部分

明确AI助手的功能定位为"helpful assistant"，负责生成基于表格数据的带编号问题列表。

### 数据表格输入部分

接收外部传入的表格数据，格式为{context_data}占位符，提供了问题生成的上下文信息基础。

### 目标指导部分

定义了生成候选问题的核心原则：问题需基于数据表中最重要的信息内容、必须可从数据表中获取答案、禁止在问题文本中提及具体数据字段或表名、当用户问题涉及多个命名实体时候选问题需全部引用。

### 示例问题部分

提供few-shot学习的示例输入，帮助模型理解期望的问题格式和风格。



## 问题及建议





### 已知问题

-   提示词硬编码在代码中，缺乏灵活性，难以动态调整提示词内容
-   没有对占位符参数（question_count、context_data）进行合法性验证，可能导致运行时错误
-   字符串拼接方式处理模板，缺少结构化定义，难以维护和扩展
-   缺乏错误处理机制，当参数缺失或类型不匹配时无法给出明确提示
-   提示词内容完全英文硬编码，缺乏国际化支持，难以适配多语言场景
-   模板内容与业务逻辑耦合，未实现配置与代码分离
-   没有日志记录，无法追踪模板的使用情况和调试问题

### 优化建议

-   将提示词模板抽取为独立的配置文件（如 JSON、YAML），实现配置与代码解耦
-   添加参数校验逻辑，验证 question_count 为正整数、context_data 非空等
-   引入模板引擎（如 Python 的 string.Template 或 Jinja2），提升模板的可维护性和安全性
-   添加错误处理和异常抛出，提供清晰的错误信息便于调试
-   设计国际化方案，支持多语言提示词模板
-   增加模板版本控制和变更日志，便于追踪和管理提示词演进
-   添加日志记录功能，追踪模板渲染过程和使用情况



## 其它




### 设计目标与约束

本模块的设计目标是为问题生成系统提供一个可配置的提示模板，支持动态生成指定数量的候选问题。约束条件包括：模板中的占位符{question_count}和{context_data}必须由调用方提供有效值，问题数量需为正整数，上下文数据需为有效的数据表格式。

### 错误处理与异常设计

由于本模块仅包含静态字符串定义，不涉及运行时错误处理。调用方在填充模板时应确保占位符被正确替换，若替换失败或替换后字符串格式异常应由调用方负责验证和处理。

### 数据流与状态机

本模块为纯数据模块，不涉及状态机设计。数据流为单向流动：调用方准备上下文数据和参数 → 格式化字符串替换占位符 → 生成最终提示词 → 传递给语言模型。

### 外部依赖与接口契约

本模块无外部依赖，仅依赖Python标准库字符串格式化功能。接口契约方面，调用方需提供question_count（整数）和context_data（字符串）两个参数，模板不支持其他自定义占位符。

### 安全性考虑

本模板设计用于与语言模型交互，需注意context_data中不应包含敏感或机密信息，因其将被嵌入提示词中传递给模型。调用方应在传入数据前进行必要的脱敏处理。

### 性能要求

本模块性能开销极低，仅涉及字符串变量定义和格式化操作，时间复杂度为O(n)，其中n为模板字符串长度。

### 配置管理

QUESTION_SYSTEM_PROMPT作为系统级提示配置，建议通过配置文件或环境变量管理，以便在不修改代码的情况下调整提示策略。后续可考虑将提示模板拆分为可插拔的模块化设计。

### 版本兼容性

当前代码使用Python 3标准字符串格式化，支持Python 3.6以上版本。如需兼容更早版本，需将f-string替换为str.format()或%格式化方式。

### 测试策略

建议为模板添加单元测试，验证：占位符替换功能正常、生成结果符合预期格式、边界条件（如question_count为0或负数）的处理逻辑正确。

### 扩展性建议

当前模板以硬编码方式定义，可考虑以下扩展方向：支持多语言提示模板、支持提示模板版本管理、支持动态加载外部模板文件、支持提示模板变量校验机制。
